⚙️ Testing Commands

Test Email Verification (Single User)

php artisan test:email salhi45adam@gmail.com --reset

Sends a verification email and displays the verification URL

Test with File-Based Mailer (No Limit)

MAIL_MAILER=file php artisan test:email test@example.com --reset

Test with any email using file-based storage (stored in storage/app/mail/)

View Stored Emails (File Mode)

ls -la storage/app/mail/

Lists all emails stored when using MAIL_MAILER=file

🔗 Monitoring API Endpoints

Available in development environment (APP_ENV=local)

Get Recent Email Logs

GET /api/debug/email/logs

Returns the 50 most recent email-related log entries

Check User Email Status

GET /api/debug/email/status/{email}

Example: /api/debug/email/status/salhi45adam@gmail.com

Returns verification status, user ID, and timestamps

Get All Test Users

GET /api/debug/email/test-users

Lists all users created with "Test Company" or "Test User" prefix

🔐 Email Verification URL

Format generated by the test command:

http://127.0.0.1:8001/api/auth/verify-email/{USER_ID}/{SHA1_HASH}?signature={SIGNED_URL_SIGNATURE}

The hash is a SHA1 of the user's email. Laravel's signed URL verification ensures the link hasn't been tampered with and is still valid (expires in 24 hours).

🔄 Testing Multiple Emails

Option 1: Use File-Based Mailer (Recommended for Testing)

MAIL_MAILER=file php artisan test:email any@email.com --reset
MAIL_MAILER=file php artisan test:email another@email.com --reset
MAIL_MAILER=file php artisan test:email third@email.com --reset

Emails will be saved to storage/app/mail/ as individual files

Option 2: Add Custom Domain to Resend (Production)

  1. Go to Resend Dashboard → Domains
  2. Click "Add Domain"
  3. Enter your domain (e.g., mail.yourdomain.com)
  4. Add DNS records as instructed by Resend
  5. Once verified, update in .env: 
    MAIL_FROM_ADDRESS="noreply@yourdomain.com"
  6. Test with any email address
